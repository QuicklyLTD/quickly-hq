# quickly-hq Documentation Hub

Bu klasor, `quickly-hq` projesinin modern yazilim ekiplerinde kullanilan dokumantasyon yapisina uygun, gorev odakli ve surdurulebilir teknik dokumantasyon setini icerir.

## Dokumanlar

- `architecture.md`: sistemin katmanlari, domain sinirlari ve veri akislari
- `api-reference.md`: route gruplari, middleware zincirleri, auth modeli ve endpoint envanteri
- `configuration.md`: ortam degiskenleri, runtime pathleri, gizli bilgi yonetimi
- `deployment.md`: build, paketleme, FTPS deploy, release ve rollback yaklasimi
- `operations-runbook.md`: canli ortam operasyonlari, gozlem, incident ve sorun giderme adimlari
- `security.md`: guvenlik resmi, riskler, hardening backlog'u
- `development-workflow.md`: yerel gelistirme, degisiklik kurallari, PR/commit beklentileri
- `evidence-index.md`: dokumantasyondaki teknik iddialarin kod referans kanitlari

## Hedef Kitle

- Backend gelistiriciler
- DevOps/SRE ekibi
- Teknik liderler ve mimarlar
- Operasyon destek ekipleri

## Dokumantasyon Ilkeleri

- Kod gercegi kaynak kabul edilir; dokumanlar kodu yansitir
- Isletme adimlari copy-paste calisabilir olacak sekilde yazilir
- Guvenlik aciklari gizlenmez, acikca backlog'a yazilir
- Basit dil, net terimler, guncellenebilir tablo/checkbox yapisi kullanilir
